# UX Design Resources

A curated collection of resources for anyone who wants to understand user experience design fundamentals — books, articles, tools, and references that shaped the content of the **"UX: What Is It? Why Does It Matter? Core Principles"** webinar.

## 📚 Books

- **[Don't Make Me Think](https://sensible.com/dont-make-me-think/)** — Steve Krug. The most accessible introduction to UX. Short, witty, and timeless.
- **[The Design of Everyday Things](https://www.nngroup.com/books/design-everyday-things-revised/)** — Don Norman. The book that introduced "affordance" and "user experience" to the design vocabulary. Written in 1988, still relevant today.
- **[100 Things Every Designer Needs to Know About People](https://www.amazon.com/Things-Designer-People-Voices-Matter/dp/0321767535)** — Susan Weinschenk. Psychology-backed design principles, each chapter 2-3 pages.
- **[Refactoring UI](https://www.refactoringui.com/)** — Adam Wathan & Steve Schoger. A design book written specifically for developers. By the creator of Tailwind CSS.
- **[101 UX Principles](https://www.amazon.com/101-UX-Principles-Definitive-Guide/dp/1788837509)** — Will Grant. Practical, opinionated UX guidelines.
- **[Interaction of Color](https://yalebooks.yale.edu/book/9780300179354/interaction-of-color/)** — Josef Albers. The definitive book on color theory and perception.

## 🎨 Design Principles & Psychology

- **[Gestalt Principles](https://www.nngroup.com/articles/principles-visual-design/)** — How the brain organizes visual information (Proximity, Similarity, Closure, Continuity, Figure/Ground).
- **[Fitts's Law](https://lawsofux.com/fittss-law/)** — Target size and distance determine interaction speed.
- **[Hick's Law](https://lawsofux.com/hicks-law/)** — More choices = slower decisions.
- **[Miller's Law](https://lawsofux.com/millers-law/)** — Working memory holds ~4 items. Use chunking.
- **[Jakob's Law](https://lawsofux.com/jakobs-law/)** — Users expect your site to work like the others they already know.
- **[Peak-End Rule](https://lawsofux.com/peak-end-rule/)** — Experiences are judged by their peak and final moments.
- **[Zeigarnik Effect](https://lawsofux.com/zeigarnik-effect/)** — Incomplete tasks are remembered better than completed ones.
- **[Doherty Threshold](https://lawsofux.com/doherty-threshold/)** — System response under 400ms feels instant.
- **[Von Restorff Effect](https://lawsofux.com/von-restorff-effect/)** — The item that stands out is the one remembered.
- **[Tesler's Law](https://lawsofux.com/teslers-law/)** — Every system has irreducible complexity. Push it behind the scenes.
- **[Aesthetic-Usability Effect](https://lawsofux.com/aesthetic-usability-effect/)** — Beautiful interfaces are perceived as easier to use.
